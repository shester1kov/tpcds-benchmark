# Project Mission

## The Problem

После завершения бенчмарка результирующий CSV-файл остаётся только локально.
Нет автоматического способа сохранить его в корпоративное S3-хранилище (Ozone S3 от ArenaData HyperWare),
что затрудняет централизованное хранение и анализ результатов.

## Who Is This For?

Для себя — развитие навыков и понимание работы с S3-совместимыми хранилищами в Go.

## Definition of Done

Проект считается завершённым, когда выполняется всё это:

- [ ] В конфиге появилась структура `S3Config` с полями: `endpoint`, `access_key`, `secret_key`, `bucket`, `use_ssl`, `enabled`
- [ ] В `config.yaml` добавлена секция `s3` с теми же полями
- [ ] В `pkg/storage/csv_storage.go` добавлен метод `GetFilePath()`, возвращающий путь к файлу
- [ ] Создан `pkg/storage/s3_storage.go` с логикой загрузки файла в Ozone S3 через `minio-go`
- [ ] В `pkg/runner/benchmark.go` после завершения бенчмарка вызывается загрузка в S3 (если `enabled = true`)
- [ ] В логах отображается статус загрузки: успех или ошибка с понятным сообщением

## Why This Matters

Централизованное хранение результатов бенчмарков позволяет сравнивать запуски во времени
и открывает возможность автоматизированного анализа без ручного копирования файлов.
